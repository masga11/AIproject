# AI Debate Arena

Веб-приложение, где два ИИ-агента автономно спорят на заданную тему в несколько раундов. Полная версия v1.0.0.

## Быстрый старт (Ollama — без регистрации)

1. Установите [Ollama](https://ollama.com/download) для Windows.
2. Откройте терминал и скачайте модель:

```bash
ollama pull llama3.2:3b
```

3. Установите зависимости проекта:

```bash
npm install
```

4. Запустите:

```bash
npm run dev
```

5. Откройте [http://localhost:5173](http://localhost:5173)

Ollama работает **локально на вашем ПК** — аккаунты, ключи и лимиты не нужны.

## Альтернатива: Groq (облако)

Если Groq заработает, в `.env`:

```bash
LLM_PROVIDER=groq
GROQ_API_KEY=gsk_ваш_ключ
```

Ключ: [console.groq.com](https://console.groq.com) → **Sign up** (не Sign in!) → через Google или GitHub.

Если видите «does not belong to any organizations» — вы нажали **Sign in** вместо **Sign up**. Нужна именно регистрация нового аккаунта.

## Возможности v1.0.0

### Агенты
- **7 встроенных агентов**: Философ, Скептик, Ученый, Политик, Экономист, Технооптимист, Гуманист
- **Конструктор кастомных агентов**: GUI для создания/редактирования/удаления своих агентов с кастомными промптами и цветами
- **Единый список выбора**: В меню дебатов встроенные и кастомные агенты в одном списке (кастомные помечены иконкой 🎨)

### Настройки моделей (Индивидуальные)
- Возможность выбрать отдельную модель и температуру (temperature) для Агента 1 и Агента 2 перед стартом
- Переключение провайдеров (Ollama/Groq) для каждого агента

### Глобальная память (RAG + SQLite)
- Сохранение истории дебатов и извлеченных знаний в БД
- Перед началом новых дебатов система ищет похожие темы в прошлом и скармливает их агентам как контекст
- Исправлен баг `NOT NULL constraint failed: messages.created_at`
- Исправлен баг, где Судья называл агентов "Философ/Скептик" вместо реальных имен

### Интерфейс и Аналитика
- Дашборд с графиками (Recharts): статистика побед, активность, сравнение моделей
- Экспорт дебатов в Markdown/JSON
- **TTS (Озвучка)**: Кнопка 🔊 у каждой реплики для голосового воспроизведения

## Как это работает

- **Frontend** (React + Vite + TypeScript) — интерфейс и стрим ответов
- **Backend** (Express) — оркестрация дебатов
- **Ollama / Groq** — генерация текста (переключается в `.env`)
- **Debate Engine** — 3 раунда, память между репликами
- **Global Memory** — SQLite база для хранения истории и знаний

| `npm run dev` | Запуск фронта и бэка |

| `npm run backend` | Только Express (порт 3002) |
| `npm run build` | Сборка фронтенда |

## Модели Ollama (умнее = дольше)

| Модель | Качество | Скорость | Скачать |
|--------|----------|----------|---------|
| `llama3.2:3b` | Базовое | Быстро | уже есть |
| `llama3.1:8b` | Выше | ~30–60 сек/реплика | `ollama pull llama3.1:8b` |
| `qwen2.5:7b` | Выше + русский | ~30–60 сек/реплика | `ollama pull qwen2.5:7b` |

На RTX 3050 Ti (4 ГБ) подойдут **8B/7B**, если готовы ждать. Выбор — в интерфейсе, поле **«Модель»**.

## Настройки (.env)

| Переменная | Описание |
|------------|----------|
| `LLM_PROVIDER` | `ollama` или `groq` |
| `OLLAMA_MODEL` | Модель по умолчанию (`llama3.2:3b`) |
| `GROQ_API_KEY` | Ключ Groq (только для `groq`) |
| `PORT` | Порт бэкенда (по умолчанию 3002) |
